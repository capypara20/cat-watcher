# 要件定義書 — ファイル常駐監視アプリケーション

| 項目 | 内容 |
|------|------|
| 文書版数 | 1.1 |
| 作成日 | 2025-07 |
| 対象システム | ファイル常駐監視アプリケーション（watcher） + CSV→TOML 変換ツール（csv2toml） |

---

## 1. システム概要

Windows 上で動作する常駐型のファイル監視アプリケーション。  
指定フォルダをリアルタイムに監視し、ファイル名やフォルダ名が命名規則（glob / 正規表現）に合致した場合に、あらかじめ定義されたアクション（コピー・移動・コマンド実行・外部プロセス起動）を自動実行する。

補助ツールとして、Excel で管理する CSV 形式のルール定義を TOML 設定ファイルへ変換する **csv2toml** を提供する。

---

## 2. 対象環境

| 項目 | 要件 |
|------|------|
| OS | Windows 10 以降（64bit） |
| ファイルシステム | NTFS（ローカルドライブ）。UNC パス（SMB）にも対応するが、OS レベルで通知が届かない環境は動作保証外 |
| 実装言語 | Rust |
| ランタイム | 追加ランタイム不要（単一バイナリ配布） |

---

## 3. 機能要件

### 3.1 ファイル監視

| ID | 要件 |
|----|------|
| F-W01 | 1 つ以上の監視対象ディレクトリをリアルタイムに監視できること |
| F-W02 | 監視イベントとして「作成・変更・削除・リネーム」を検知できること |
| F-W03 | ファイルおよびフォルダの両方を検知対象にできること（`target` で file / directory / both を選択） |
| F-W04 | サブディレクトリの再帰監視を ON/OFF で切り替えられること |
| F-W05 | glob パターンまたは正規表現でファイル名・フォルダ名をフィルタリングできること（排他。どちらか一方を指定必須） |
| F-W06 | 除外パターン（`exclude_patterns`）により特定のファイル名を除外できること |
| F-W07 | Windows の隠しファイル・隠しフォルダ（`FILE_ATTRIBUTE_HIDDEN`）をルール単位で検知対象に含める/除外する設定（`include_hidden`）を提供すること |

### 3.2 アクション

| ID | 要件 |
|----|------|
| F-A01 | 検知したファイル/フォルダを指定先ディレクトリへ**コピー**できること |
| F-A02 | 検知したファイル/フォルダを指定先ディレクトリへ**移動**できること |
| F-A03 | 任意のシェルコマンドを**実行**できること（cmd / PowerShell / pwsh から選択） |
| F-A04 | 任意の実行ファイルをシェルを介さず**直接起動**できること |
| F-A05 | 1 つのイベントに対して複数のアクションを順次実行できること（アクションチェーン） |
| F-A06 | コピー/移動時にサブディレクトリ構造の維持/フラット化を選択できること |
| F-A07 | コピー/移動時に既存ファイルの上書き/スキップを選択できること |
| F-A08 | コピー/移動後にハッシュ値比較（BLAKE3）でファイルの完全性を検証できること。ルール単位で有効/無効を選択可能 |

### 3.3 設定

| ID | 要件 |
|----|------|
| F-C01 | 設定ファイルは TOML 形式であること |
| F-C02 | グローバル設定（`global.toml`）と監視ルール（`rules.toml`）の 2 ファイル構成であること |
| F-C03 | すべての設定項目にデフォルト値を設けず、明示的な指定を必須とすること |
| F-C04 | 起動時に設定のバリデーションを行い、不正な場合はエラー終了すること |
| F-C05 | CLI 引数で設定ファイルパス、ドライラン、ログレベル上書き、バリデーション専用モードを指定できること |

### 3.4 CSV→TOML 変換ツール

| ID | 要件 |
|----|------|
| F-T01 | BOM 付き UTF-8 の CSV ファイルを入力として受け付けること |
| F-T02 | CSV の各行を TOML のルール定義に変換し出力できること |
| F-T03 | 同一ルール名の複数行をアクションチェーンとして統合できること |
| F-T04 | 変換時にバリデーション（必須項目、型、排他制約など）を実行できること |
| F-T05 | ドライラン（stdout 出力のみ）およびバリデーション専用モードを提供すること |

### 3.5 ロギング

| ID | 要件 |
|----|------|
| F-L01 | JSON 形式の構造化ログをファイル出力できること |
| F-L02 | ログローテーション（日次 / なし）を選択できること |
| F-L03 | ログレベル（trace / debug / info / warn / error）を設定できること |

### 3.6 運用制御

| ID | 要件 |
|----|------|
| F-O01 | Ctrl+C 等のシグナルでグレースフルシャットダウンできること |
| F-O02 | ドライランモードでアクションを実行せずにログのみ出力できること |
| F-O03 | 起動時に監視対象内の既存ファイル/フォルダに対してルール評価を実行すること |

## 4. 非機能要件

### 4.1 性能

| ID | 要件 | 目標値 |
|----|------|--------|
| NF-P01 | アイドル時 CPU 使用率 | < 0.1% |
| NF-P02 | アイドル時メモリ使用量 | < 10 MB |
| NF-P03 | イベント検知〜アクション実行開始の遅延 | < 1 秒（デバウンス 500ms 含む） |

### 4.2 信頼性

| ID | 要件 |
|----|------|
| NF-R01 | ファイルロック（書き込み中のファイル）に対してリトライ機構を持つこと |
| NF-R02 | アクション失敗時にリトライ（回数・間隔は設定可能）を行えること |
| NF-R03 | OS バッファ溢れ時にフルスキャンで復旧できること |
| NF-R04 | 監視ディレクトリが消失した場合にエラーログを出力し終了すること |
| NF-R05 | コピー/移動後のハッシュ検証は大容量ファイル（数 GB 以上）でも高速に動作すること（BLAKE3 アルゴリズムを使用） |

### 4.3 安全性

| ID | 要件 |
|----|------|
| NF-S01 | 循環参照（watch_path ↔ destination）を起動時に検出しエラーとすること |
| NF-S02 | シンボリックリンクを追跡しないこと |
| NF-S03 | コピー/移動先パスを正規化し、パストラバーサルを防止すること |
| NF-S04 | 設定ファイルの NTFS 権限管理により、不正な設定変更を防御する運用とすること |

### 4.4 保守性

| ID | 要件 |
|----|------|
| NF-M01 | 設定変更時はアプリ再起動により反映する（ホットリロードは実装しない） |
| NF-M02 | 単一バイナリとして配布可能であること |

### 4.5 対応ファイル名

| ID | 要件 |
|----|------|
| NF-F01 | 日本語ファイル名（マルチバイト文字）を動作保証すること |
| NF-F02 | 半角スペース・全角スペースを含むファイル名を動作保証すること |
| NF-F03 | 260 文字超の長いパスに `\\?\` プレフィクスで対処すること |

## 5. 制約事項

| 項目 | 内容 |
|------|------|
| OS | Windows 限定（Linux / macOS は将来の拡張） |
| ネットワーク | SMB で OS 通知が届かない環境は動作保証外 |
| 設定ファイルのセキュリティ | コマンドインジェクション対策はサニタイズではなく NTFS 権限管理に委ねる |
| ホットリロード | 実装しない。設定変更は再起動で反映 |
| command / execute の完了管理 | fire-and-forget。タイムアウト・終了コード判定・stdout/stderr のログ記録は行わない |
